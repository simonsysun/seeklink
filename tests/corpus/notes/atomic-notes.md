# Atomic notes

An atomic note captures **one idea, self-contained**. You should be able
to read it without context and understand it. No "see also for background"
hand-waves; if background is needed, quote or paraphrase it in the note
itself, or defer to an explicit wikilink the reader can click.

The test: can a stranger, given only this note, understand its single
claim? If yes, it's atomic. If they need three other notes to make sense
of it, it isn't — split.

## Why atomicity matters

- **Linking**: you can only usefully link to a note if the note has a
  clear subject. "My thoughts on memory" is unlinkable; "Forgetting
  curves are empirically power-law, not exponential" is linkable.
- **Reuse**: atomic notes compose into arguments. Long monolithic notes
  stay stuck in the context they were written for.
- **Progressive summarization**: each atomic note can be distilled into
  a flashcard and fed to SRS (see [[note-to-card-workflow]]). Non-atomic
  notes resist this distillation.

See [[zettelkasten]] for the methodological origin, and
[[note-to-card-workflow]] for how atomic notes feed [[spaced-repetition]].

## What atomic is *not*

- **Not short**. Length isn't the point. A 2000-word essay on one
  specific claim, densely argued, is still atomic. A 40-word paragraph
  that vaguely gestures at three topics is not.
- **Not a definition**. "FSRS is an algorithm" is not an atomic note, it's
  a label. "[[FSRS]] achieves ~30% lower review workload than SM-2 at
  equal retention because it models population-fitted stability rather
  than per-card ease" is an atomic note — it's a claim, falsifiable, with
  reasoning.
- **Not orphan**. Atomic ≠ isolated. An atomic note still has context via
  links. It just contains its own meaning.
