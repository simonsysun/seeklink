## 2026-W18 — 关于 RAG pipeline 的几个碎片

random notes this week, 没打算写成完整论证。

### 1. Reranker 不是"让结果更准"，是"让结果更稳定"

上周一直在想为什么 reranker 的体感改进比纸面 nDCG 数字大。答案：
nDCG 测的是平均，人体感受的是**最差几次**。reranker 降低了 tail
latency 上的坏 case 频率——不是把 top-1 变得更准，是把 top-1 被
替成明显不相关结果的概率降低。

这个 insight 有点像 SLO 思路：reliability > average。

### 2. Query expansion ≈ 用户搜索前的多角度自问自答

Agent 自己会做。人不会。这就是为什么笔记软件 UI 需要内置 expansion，
CLI agent 场景可以跳过。见 [[retrieval-augmented-generation]]。

### 3. CJK 处理上 seeklink 的 unfair advantage

国内做 RAG 的产品都用 openai 的 embedder 或者 BGE-large。jina-v2-zh
是 2024 年做中文 embedder 做得最细的一家，但英文场景少人用所以没
listicle 报道。对 CJK-first vault 是 underappreciated 的好东西。
见 [[vector-embeddings]]。

### 4. Hybrid retrieval 的论文很多但工具很少

IR 会议论文几乎都在做 hybrid + rerank，但 pip install 能用的
end-to-end 工具寥寥无几。seeklink 的定位就站这个缝里：论文早有，
好用的 CLI 罕见。
