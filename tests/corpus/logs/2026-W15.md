## 2026-W15 周记

这周主题：[[agent-memory-patterns]]。为什么 Claude / Cursor / 本地 agent 都
在这个问题上卡住——不是模型太弱，是 retrieval 太差。

### 讨论记录

和 C 讨论了一个小时。主要 insight：

1. LLM 本身 stateless，所有"记忆"都是 retrieval。
2. Retrieval 质量 = agent 智商上限。返回错的 chunk = agent 自信地胡说。
3. 现有方案多是 vector-only RAG，对 CJK 极差，对 exact-keyword 极差。
4. 没人认真做 hybrid + rerank 这套工业界 IR 标配。

### 这个项目的立意

seeklink 的差异化不是"又一个语义搜索"，是**把 IR 工业界做了 20 年的
最佳实践带到个人 vault + agent 场景**：
- BM25 + vector + graph + title 四路召回（[[RRF]] 融合）
- 可选 cross-encoder 重排（[[attention-mechanism]]）
- CJK 一等公民（jieba tokenizer，见 [[sqlite-fts5-cjk]]）
- 全本地，无 API，CLI-first

### 技术债记录

- cold-start 路径没传 reranker（和 daemon 行为不一致）—— 得修
- line-range retrieval 缺失，agent 用起来浪费 context
- title 权重 1.5 是 v0.2 拍的，没 A/B 过
- reranker 的 yes/no sigmoid 直接替换 RRF 分数，exact alias 硬命中会被
  demote——这是 v0.3 Item 1 要解决的
